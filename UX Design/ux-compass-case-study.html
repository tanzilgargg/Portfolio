UX Compass

A comprehensive mobile-first learning platform designed to help UX/design students navigate structured learning paths, practice real projects, and develop portfolio-ready deliverables.

My Role

UX Designer & Researcher

Duration

6 weeks

Platform

Mobile-First Application

Project Overview

UX Compass is a comprehensive mobile-first learning platform designed to help UX/design students (ages 18–25) navigate structured learning paths, practice real projects, and develop portfolio-ready deliverables. The platform combines step-by-step project guidance, curated resources, gamification elements, and a reflection system into a cohesive, engaging experience.

The project includes a complete design system, five core screens (Onboarding, Dashboard, Project Stage, Resources, Portfolio), reusable components, and a responsive mobile-first layout.

Problem + Goals

Problem

UX students often struggle with fragmented learning resources, unclear project expectations, and difficulty translating theory into actionable portfolio work. Existing platforms fail to guide users through the complete UX process in a structured and motivating way, leading to low confidence and inconsistent outcomes.

Goals

  • Build a single, coherent system for UX learning and project execution
  • Reduce cognitive overload through clear visual hierarchy and progressive disclosure
  • Incorporate gamification and progress tracking to encourage consistent engagement
  • Provide reflection and portfolio-building tools to foster design reasoning
  • Support accessibility and mobile-first usability for students on the go

Research Plan

To understand the needs and challenges of UX students, I conducted comprehensive research using multiple methodologies to gather both qualitative and quantitative insights.

Methodologies

  • Semi-structured interviews with beginner/intermediate UX students
  • Online surveys to gather broader quantitative insights
  • Competitive analysis of existing UX learning apps and resources

Questions Asked

  • How do you currently learn UX concepts and methods?
  • What are the biggest challenges in completing UX projects?
  • How do you understand portfolio expectations?
  • What motivates you to complete learning tasks consistently?
  • What features would make a learning app useful and engaging?

Findings

User Quotes

"I spend more time figuring out what to do next than actually designing."

"Portfolio requirements feel unclear until it's too late."

Pain Points

  • Fragmented and unstructured learning materials
  • Difficulty translating theory into projects
  • Low confidence in UX decision-making
  • Minimal feedback or reflection opportunities

Emotional Themes

  • Overwhelm and anxiety around learning and portfolio creation
  • Frustration with scattered guidance
  • Desire for clarity, reassurance, and structured progress

Personas + Empathy Map

Persona

Name: Maya, 21, UX design student

Needs: Step-by-step guidance, clear expectations, portfolio-ready output

Behaviors: Searches online for tutorials, switches between platforms, often unsure of next steps

Empathy Map Highlights

  • Think: "Am I learning UX correctly?"
  • Feel: Anxious, overwhelmed, unmotivated without structure
  • Say: "I'm not sure how to start this project."
  • Do: Jump between resources, take notes, but rarely complete full projects

Insights & Opportunities

  • Students need guided, structured pathways, not more scattered content
  • Reflection and rationale improve learning retention
  • Progress tracking and gamification can enhance engagement
  • Translating industry and academic expectations into actionable steps builds confidence
  • Bite-sized tasks with clear completion markers reduce cognitive load

Brief User Flow

Onboarding → Dashboard → Project Stages → Resources → Portfolio → Profile

Wireframes

Five core screens with reusable components and mobile-first design. Progressive disclosure of information, expandable task cards, horizontal scrolling carousels, and touch-friendly interactions.

Reflection + Lessons Learned

Designing UX Compass reinforced that effective UX education requires clarity, structure, and engagement. Students benefit from guided learning rather than self-directed exploration alone.

Key takeaways:

  • Gamification and progress visualization increase consistent engagement
  • Reflection prompts encourage deeper understanding of design reasoning
  • Building a robust component library improves scalability and consistency
  • Mobile-first, accessible design ensures usability for all students
  • Iterative testing with target users is essential for balancing guidance with autonomy

UX Compass demonstrates the value of designing educational tools with the user at the center, integrating learning, doing, and reflecting into a seamless experience.
